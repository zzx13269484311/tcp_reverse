================================================================================
                        文件反转服务程序运行说明
================================================================================

一、程序简介

本程序包含两个 Python 脚本：

  reversetcpserver.py   - 多线程 TCP 服务端，接收客户端连接，对文本块进行字符反转。
  reversetcpclient.py   - 命令行客户端，分块发送 ASCII 文本文件，接收反转结果，
                并最终输出整体反转后的文件。

工作原理：原始文件 -> 随机分块 -> 每个块内字符反转 -> 所有块顺序反转 -> 输出文件。

二、运行环境

  - 操作系统：Windows / Linux / macOS
  - Python 版本：3.6 或更高
  - 依赖库：仅使用 Python 标准库，无需额外安装

三、文件列表

  reversetcpserver.py          - 服务端程序
  reversetcpclient.py          - 客户端程序
  run_log.txt        - 程序运行时自动生成的日志文件
  source.txt          - 用户准备的输入文件
  reversed_output.txt         - 客户端生成的输出文件

四、配置选项

【服务端配置】

  服务端默认监听端口 8888。如需修改，请编辑 reversetcpserver.py 最后一行：
      start_server(8888)   → 改为需要的端口号，例如 start_server(9000)

【客户端配置】

  客户端通过命令行参数进行配置，格式如下：

  python client.py <server_ip> <server_port> <Lmin> <Lmax> <seed> <src_filename> <out_filename>

  参数说明：
    server_ip      - 服务端 IP 地址，例如 127.0.0.1
    server_port    - 服务端端口号，必须与服务端一致
    Lmin           - 每个分块的最小字节数（整数，>=1）
    Lmax           - 每个分块的最大字节数（整数，>=Lmin）
    seed           - 随机数种子（整数），用于重现分块大小序列
    src_filename   - 输入文件名（必须是 ASCII 编码的文本文件）
    out_filename   - 输出文件名（程序将最终反转结果写入该文件）

  示例：
    python client.py 127.0.0.1 8888 5 10 42 input.txt output.txt

五、通信协议（简要）

  所有多字节整数使用网络字节序（大端，!H 和 !I）。

  报文类型：
    1 - Initialization  (客户端 -> 服务端): 2字节Type + 4字节总块数N
    2 - Agree           (服务端 -> 客户端): 2字节Type
    3 - reverseRequest  (客户端 -> 服务端): 2字节Type + 4字节数据长度 + 原始文本块
    4 - reverseAnswer   (服务端 -> 客户端): 2字节Type + 4字节数据长度 + 反转后文本块

  交互流程：
    1) 客户端发送 Type=1（含N）
    2) 服务端回复 Type=2
    3) 对每个块（共N块）：
        客户端发送 Type=3（原始数据）
        服务端回复 Type=4（反转后数据）
    4) 客户端将收到的反转块按相反顺序拼接，写入输出文件。

六、运行步骤

  1. 准备输入文件
     创建一个 ASCII 编码的文本文件（例如 input.txt），内容仅包含英文字母、数字、标点。
     不支持中文或其他非 ASCII 字符。

  2. 启动服务端
     打开一个命令行终端，执行：
         python server.py
     看到输出 "服务端启动，监听0.0.0.0:8888" 表示成功。
     服务端会生成 run_log.txt 记录通信日志。

  3. 运行客户端
     打开另一个终端，执行例如：
         python client.py 127.0.0.1 8888 5 10 42 input.txt output.txt
     客户端会显示每个块的反转结果，最终输出 "全部块处理完毕，反转文件已输出至 output.txt"。

  4. 验证结果
     用文本编辑器打开 output.txt，检查内容是否符合预期：
       原始文件分块 -> 每块字符反转 -> 块列表反转 -> 拼接。

  5. 并发测试（可选）
     服务端支持多客户端同时连接。可以在多个终端中几乎同时运行多个客户端，
     测试服务端的并发处理能力。服务端会为每个客户端创建独立线程。

七、日志文件

  - 服务端和客户端各自生成 run_log.txt，记录时间戳、报文方向、类型、长度及内容摘要。
  - 由于多线程写入，不同客户端的日志记录可能交错，不影响功能。
  - 服务端启动时会清空现有日志文件。

八、注意事项

  1. 文件编码：程序强制使用 ASCII 编码。输入文件若包含非 ASCII 字符（如中文）将报错。
  2. 分块规则：每个块大小在 [Lmin, Lmax] 范围内随机，最后一个块可能小于 Lmin。
  3. 端口占用：如果 8888 端口被占用，请修改服务端端口并相应修改客户端命令。
  4. 防火墙：如果服务端和客户端不在同一台机器，需确保服务端端口允许外部访问。
  5. 并发性能：服务端每连接创建一个线程，适合中小规模并发测试（建议 <100 连接）。
  6. 异常处理：网络异常或协议错误会导致连接关闭，错误信息写入日志。

九、常见问题

  Q: 客户端报错 ConnectionRefusedError
  A: 服务端未启动或端口号不一致，请先运行 server.py 并核对端口。

  Q: 服务端日志出现“异常报文”
  A: 客户端发送了不符合协议的数据，请检查命令行参数及输入文件格式。

  Q: 输出文件内容与原文件完全不同
  A: 程序执行的是“块内反转+块顺序反转”，并非全文直接反转。可用小文件（如 "abcd"，Lmin=Lmax=2）手动验证逻辑。

  Q: 能否处理二进制文件？
  A: 不建议。本程序针对 ASCII 文本设计，处理二进制会因解码错误而失败。
